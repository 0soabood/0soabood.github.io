I wanted a place to write. Not a newsletter, not a Twitter thread — just a quiet corner of the internet where I can think out loud about things I'm building and learning.

This post is about how I set up this blog: the stack I chose, the few gotchas I ran into, and why the whole thing took less than an afternoon.

## Why Docusaurus?

The requirements were simple: write in Markdown, deploy to GitHub Pages for free, and not spend two weekends configuring webpack. I looked at a few options — Jekyll, Hugo, Astro, 11ty — and landed on **Docusaurus** for a few reasons:

1. It's maintained by Meta's open source team and has active development
2. It supports **MDX** out of the box, so I can drop React components into posts if I ever want to
3. Blog-only mode means I get a clean blog without the docs sidebar I don't need
4. It handles RSS/Atom feeds, reading-time estimates, and author profiles without plugins

## Scaffolding the Project

```bash
npx create-docusaurus@latest my-blog classic
cd my-blog
npm install
```

The classic preset gives you docs + blog. Since I only want the blog, I needed to turn off the docs section and make the blog serve from the root path.

## Blog-Only Mode

In `docusaurus.config.js`, inside the `presets` array, change the blog plugin config:

```js
presets: [
  [
    'classic',
    {
      docs: false,  // disable the docs plugin entirely
      blog: {
        routeBasePath: '/',  // serve blog from the site root
        showReadingTime: true,
        feedOptions: {
          type: ['rss', 'atom'],
          xslt: true,
        },
        onInlineTags: 'warn',
        onInlineAuthors: 'warn',
        onUntruncatedBlogPosts: 'warn',
      },
      theme: {
        customCss: './src/css/custom.css',
      },
    },
  ],
],
```

With `docs: false` and `routeBasePath: '/'`, the homepage is now your blog listing. Clean.

## GitHub Pages Configuration

This is where I hit the only real snag. The scaffold generates placeholder values you need to update before deploying:

```js
// What the scaffold generates (wrong for my repo):
url: 'https://Lorchissimo.github.io',
organizationName: 'Lorchissimo',
projectName: 'Lorchissimo.github.io',

// What it should be:
url: 'https://0soabood.github.io',
organizationName: '0soabood',
projectName: '0soabood.github.io',
```

Also worth adding `trailingSlash: false` explicitly — without it you'll see a warning about inconsistent trailing slash behavior during deployment, which can cause issues with some hosting setups:

```js
trailingSlash: false,
```

## Authors and Tags

Docusaurus supports a structured authors file. Create `blog/authors.yml`:

```yaml
abood:
  name: 0soAbood
  image_url: https://github.com/YourUsername.png
  socials:
    github: YourUsername
```

And `blog/tags.yml` for consistent tag metadata:

```yaml
general:
  label: General
  permalink: /general
  description: General posts
```

Then in any post's frontmatter:

```yaml
---
title: "My Post Title"
date: 2026-05-03
authors: abood
tags: [general]
---
```

## Dark Mode

One line in `themeConfig`:

```js
themeConfig: {
  colorMode: {
    respectPrefersColorScheme: true,
  },
  // ...
}
```

This makes the blog respect the user's OS-level dark/light preference automatically. No toggle required.

## The Result

The whole setup — scaffolding, configuration, writing the first post, and getting it ready to deploy — took a few hours. The project structure is minimal:

```
my-blog/
├── blog/
│   ├── authors.yml
│   ├── tags.yml
│   └── 2026-05-03-first-post.mdx
├── src/
│   └── css/
│       └── custom.css
├── static/
│   └── img/
└── docusaurus.config.js
```

No database, no CMS, no server. Just Markdown files committed to a repo that deploys automatically via GitHub Actions.

## Key Takeaways

- Set `docs: false` and `routeBasePath: '/'` to get a blog-only Docusaurus site
- Always update the `url`, `organizationName`, and `projectName` fields in the config before deploying
- Add `trailingSlash: false` to avoid GitHub Pages redirect inconsistencies
- `authors.yml` and `tags.yml` keep your post metadata consistent across posts
- `respectPrefersColorScheme: true` gives you free dark mode with zero extra work
